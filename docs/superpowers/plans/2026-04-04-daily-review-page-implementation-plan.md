# 每日回顾页面实施计划

日期：2026-04-04

## 目标

实现一个独立的“每日回顾”页面，用于展示今天最值得用户回看的旧内容，并支持详情侧栏查看。

## 阶段 1：页面骨架与路由

### 目标

- 新增页面路由
- 新增左侧导航入口
- 新增页面基础布局

### 任务

- 新建 `DailyReviewPage`
- 更新路由配置
- 更新侧边栏导航
- 先搭出页面区块骨架：
  - 今日回顾概览
  - 今日推荐回顾
  - 最近主题聚焦
  - 已沉淀进知识图谱

### 验证

- 页面能正常访问
- 导航点击正常
- 布局无报错

## 阶段 2：后端回顾数据接口

### 目标

- 提供每日回顾页面所需数据

### 任务

- 新增回顾数据 schema
- 新增每日回顾 service
- 新增回顾 API

### 后端输出建议

- 概览统计
- 推荐回顾卡片列表
- 最近主题聚焦列表
- 最近已入图谱内容列表

### 验证

- API 返回结构稳定
- 空数据情况下也能返回合理结果

## 阶段 3：推荐逻辑

### 目标

- 实现“今日推荐回顾”规则评分

### 候选池

- 最近 14 天新增记忆
- 最近进入知识图谱的记忆
- 与最近高频主题有关的记忆

### 评分规则

- `+3` 已进入知识图谱
- `+3` 属于最近高频主题
- `+2` 最近 7 天新增
- `+2` 与最近新增内容相关
- `+1` 摘要较完整
- `-1` 内容过短

### 验证

- 推荐结果数量合理
- 推荐原因能解释

## 阶段 4：前端卡片展示

### 目标

- 用卡片式方式展示推荐回顾与主题聚焦

### 任务

- 设计推荐卡片组件
- 设计主题卡组件
- 设计已入图谱卡组件

### 卡片最少信息

- 标题
- 摘要
- 状态标签
- 推荐原因

### 验证

- 页面不拥挤
- 信息密度适中
- 推荐原因可读

## 阶段 5：详情侧栏

### 目标

- 点击卡片后，在右侧打开一个专门的回顾详情侧栏

### 任务

- 新增详情侧栏组件
- 支持点击卡片打开
- 展示：
  - 标题
  - 时间
  - 推荐原因
  - 摘要
  - 原文
  - 图谱状态
  - 主题标签

### 验证

- 侧栏开关正常
- 不影响页面主区域浏览

## 阶段 6：与知识画像联动

### 目标

- 复用 knowledge profile 中的最近主题 / 重点信息

### 任务

- 从最新 knowledge profile 读取 `major_topics`
- 从最新 knowledge profile 读取 `recent_focuses`
- 将其用于主题聚焦和推荐辅助

### 验证

- 页面主题与当前知识画像一致性更高

## P0 范围

- 页面路由
- 回顾 API
- 规则评分推荐
- 推荐卡片
- 最近主题聚焦
- 已入图谱区块
- 详情侧栏

## P1 范围

- 待继续整理区块
- 标记已回顾
- AI 生成每日回顾摘要
- 更复杂的图谱关联推荐

## 建议顺序

1. 页面骨架与路由
2. 后端 API
3. 规则评分逻辑
4. 前端卡片区块
5. 详情侧栏
6. 知识画像联动

## 验收标准

- 用户可在新页面查看“今日推荐回顾”
- 推荐内容不是纯时间列表，而是有规则依据
- 推荐原因可见
- 点击卡片后可在右侧查看详情
- 页面可与当前知识画像信息形成联动
