# 每日回顾页面设计

日期：2026-04-04

## 背景

当前项目已经具备：

- 记忆录入与管理
- 记忆提交到知识图谱
- 基于知识图谱的 Agentic RAG
- 自动生成的知识画像 overlay

但用户在“今天打开系统”时，还缺少一个适合复盘与复习的入口。现有页面更偏向：

- 维护记忆
- 上传资料
- 知识对话
- 查看图谱

它们不直接回答：

- 最近哪些旧内容值得今天回看
- 哪些内容已经沉淀进知识图谱
- 最近主要在积累哪些主题
- 哪些记录适合今天继续整理

因此需要新增一个“每日回顾”页面，作为用户的知识复盘工作台。

## 目标

新增一个独立页面“每日回顾”，在用户打开页面时，优先展示：

- 今天最值得回顾的旧内容
- 最近几天的重要主题聚焦
- 最近已经沉淀进知识图谱的重点内容
- 仍值得继续整理的内容

页面定位不是“今天新增了什么”，而是“今天推荐你回顾什么”。

## 非目标

第一版不做：

- 日历视图
- 热力图
- 复杂统计图表
- 全文长列表回顾
- 任务管理系统
- AI 生成的整页长日报

## 页面定位

页面名称建议：

- 每日回顾

页面定位：

- 单独新页面
- 偏知识复盘与复习
- 不是操作后台，也不是简单列表页

## 页面结构

### 1. 今日回顾概览

展示轻量总览信息，例如：

- 今日推荐回顾条数
- 最近 7 天新增记忆数
- 最近进入知识图谱的条数
- 当前最活跃主题

目的：

- 让用户一眼看懂最近知识活动情况

### 2. 今日推荐回顾

这是页面主区域。

展示 3 到 6 张“今天最值得回顾”的卡片。

每张卡片包含：

- 标题
- 2 到 3 行摘要
- 标签
  - 已入图谱
  - 最近高频主题
  - 最近新增
  - 与近期内容相关
- 推荐原因短句

这是每日回顾页面最核心的推荐区域。

### 3. 最近主题聚焦

按主题聚合最近知识，而不是按时间流水展示。

每个主题卡展示：

- 主题名
- 相关记忆数量
- 最近一次更新时间
- 一句简短总结

示例主题：

- UI 与 UX
- 冒烟测试与 API 模拟
- 垃圾分类投放安排

### 4. 已沉淀进知识图谱

展示最近已成功进入知识图谱的内容。

目标：

- 让用户知道哪些知识已经真正进入系统可检索认知

### 5. 待继续整理

展示那些：

- 已记录但尚未入图谱
- 内容零散但值得补充
- 与最近高频主题相关但还未充分沉淀

目标：

- 引导用户下一步继续补充和整理

## 推荐逻辑

### 推荐策略

第一版采用“规则评分推荐”，而不是纯时间排序或纯模型排序。

推荐理念：

- 最近记录的内容优先
- 已进入知识图谱的内容优先级更高
- 但两者混合，而不是只看某一侧

### 候选池

从以下内容里构建候选：

- 最近 14 天新增的记忆
- 最近进入知识图谱的记忆
- 与最近高频主题有关的记忆

### 评分因子

建议第一版权重如下：

- `+3` 已进入知识图谱
- `+3` 属于最近高频主题
- `+2` 最近 7 天新增
- `+2` 与最近新增内容相关
- `+1` 摘要较完整、适合回顾
- `-1` 内容过短、信息密度过低

排序后选择前 3 到 6 条作为“今日推荐回顾”。

### 推荐原因展示

前端卡片需要把推荐原因展示出来，例如：

- 已入图谱
- 最近高频主题
- 最近新增
- 与近期内容相关

这样用户能够理解为什么系统推荐该条内容。

## 详情交互

点击“今日推荐回顾”卡片后，不复用现有记忆管理详情弹窗，而是打开一个专门为每日回顾设计的右侧详情侧栏。

### 详情侧栏内容

- 标题
- 时间
- 状态标签
- 为什么今天推荐回顾它
- 内容摘要
- 原始内容
- 相关主题标签
- 是否已进入图谱

### 底部操作

建议第一版预留但保持克制：

- 去记忆管理查看
- 继续提问知识库
- 重新提交到知识图谱（如果尚未入图谱）

## 与知识画像的关系

“每日回顾”与“Agent 知识画像 overlay”共享一部分统计基础。

可复用内容包括：

- 最近高频主题
- 最近新增重点
- 已入图谱内容统计

这样可以减少重复计算，并让：

- Agent 对知识库的理解
- 用户对最近知识积累的理解

使用同一套底层信号。

## 数据来源

第一版优先使用：

- Memory 表
- `graph_status`
- `created_at`
- `graph_added_at`
- 记忆标题 / 摘要 / 原文
- 已有 knowledge profile 表

第一版暂不直接做复杂图谱传播分析。

## 视觉与布局建议

风格保持当前项目较克制的知识工作台风格：

- 卡片式布局
- 清晰留白
- 轻描边
- 小面积状态标签
- 不使用高饱和大面积强调色

重点是“像复盘工作台”，不是“数据后台”。

## P0

- 新增“每日回顾”页面路由与侧边栏入口
- 构建回顾候选池
- 实现规则打分推荐
- 展示今日推荐回顾卡片
- 展示最近主题聚焦
- 展示最近已入图谱内容
- 实现右侧详情侧栏

## P1

- 展示可继续整理内容
- 支持“标记今天已回顾”
- 引入 AI 总结的今日回顾摘要
- 丰富推荐原因
- 做更强的主题聚合与关联分析

## 风险与注意事项

- 推荐逻辑过于机械会让页面像“普通列表”
- 推荐逻辑过于黑盒会让用户不理解为什么被推荐
- 若卡片内容过长，页面会失去“复盘”节奏
- 第一版应先强调可解释性，而不是复杂智能

## 结论

“每日回顾”第一版应作为一个独立页面存在，并围绕“今天推荐你回顾什么”展开，而不是展示“今天新增了什么”。

最核心的价值在于：

- 推荐有理由
- 页面有复盘感
- 与知识图谱和知识画像形成联动
