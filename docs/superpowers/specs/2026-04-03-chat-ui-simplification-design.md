# Chat UI 简化设计文档

## 概述
将当前的聊天界面进行简化，使AI回复更加简洁直观，参考目标设计风格，去除AI消息的视觉包裹，优化思考过程的展示方式。

## 目标
1. 去除AI消息的Paper包裹（背景色、边框、阴影），使AI回复直接显示在页面上
2. 简化思考过程的展示，折叠时只显示单行文字，展开时显示简洁的时间线
3. 保持用户消息的现有气泡样式不变
4. 保留所有现有功能（实时更新、状态追踪、预览项等）

## 设计细节

### 1. 消息布局结构

#### 用户消息（保持不变）
- 右对齐
- Paper组件包裹
- 带背景色（primary.main）
- 带边框和阴影
- 圆角气泡样式

#### AI消息（重构）
结构从上到下：
1. 思考过程组件（ThinkingProcess）
2. AI回复内容（MarkdownContent / AssistantContent）
3. 引用列表（CitationList，已内联在内容中）

移除元素：
- ~~Paper包裹~~
- ~~AI标签文字~~
- ~~独立的引用列表展示~~（引用已通过CitationInline内联在内容中，CitationList在内容末尾展示）

### 2. 思考过程组件（ThinkingProcess）

#### 组件职责
- 显示AI的思考和执行过程
- 支持折叠/展开交互
- 实时更新思考状态
- 显示执行步骤的详细信息

#### 折叠状态（默认）
视觉样式：
- 单行文字显示，无背景，无边框
- 字体大小：caption (12px)
- 颜色：text.secondary
- 右侧展开图标（▼）
- 文字过长时使用省略号截断

显示逻辑：
- 如果有多个步骤，只显示最新的一步（timeline数组的最后一项）
- 显示格式：
  - 进行中："{当前步骤描述}... ▼"
  - 已完成："{最后步骤描述} · 共{N}步 ▼"
- 示例：
  - "正在组织最终回答... ▼"
  - "检索第3轮 · 共3步 ▼"

交互：
- 整行可点击
- 点击后展开显示完整时间线
- 鼠标悬停时显示手型光标

#### 展开状态
视觉样式：
- 垂直时间线布局
- 无背景色（透明）
- 简洁的步骤列表
- 左侧时间线指示器（竖线 + 圆点）

显示内容：
- 所有执行步骤按顺序展示
- 每个步骤包含：
  - 步骤标题（如"检索第1轮"）
  - 详细描述
  - 状态图标（✓ 完成 / ⋯ 进行中 / ✗ 错误）
  - 预览项（如检索到的证据，使用Chip展示）
  - 预览溢出提示（如"+5条证据"）

时间线样式：
- 左侧垂直线：1.5px宽，淡灰色渐变
- 步骤圆点：14px直径，根据状态显示不同颜色
  - 完成：灰色，带✓图标
  - 进行中：橙色，带⋯图标，带动画
  - 错误：红色，带✗图标

交互：
- 点击折叠按钮（▲）收起
- 展开/收起使用Collapse动画（200ms）

### 3. 数据流和状态管理

#### 输入数据
- `timelineEvents: ChatTimelineEvent[]` - 实时更新的时间线事件
- `trace: AgentTrace | null` - Agent执行轨迹
- `active: boolean` - 是否正在执行中

#### 内部状态
- `expanded: boolean` - 展开/折叠状态
- `typedDetail: string` - 占位符动画的打字效果文本

#### 核心逻辑（保留）
- `buildThinkingTimeline()` - 构建时间线数据
  - 优先使用timelineEvents
  - 回退到trace.tool_loop
  - 添加占位符步骤（如果active且无当前步骤）
- 占位符动画效果（打字机效果）
- 实时状态更新

### 4. 组件层级变化

#### 当前结构
```
ChatMessageList
  └─ Paper (AI消息容器)
      ├─ Typography ("AI"标签)
      ├─ ThinkingSummaryPanel (复杂的Paper组件)
      ├─ AssistantContent
      └─ CitationList
```

#### 新结构
```
ChatMessageList
  └─ Box (AI消息容器，无样式)
      ├─ ThinkingProcess (轻量组件)
      ├─ AssistantContent
      └─ CitationList (保留，因为在内容末尾展示)
```

### 5. 样式规范

#### 颜色
- 思考过程文字：text.secondary (#6f6a61)
- 进行中状态：secondary.main (橙色 #d97757)
- 完成状态：text.secondary
- 错误状态：error.main (红色)
- 时间线：rgba(176,174,165,0.4) 渐变到 rgba(176,174,165,0.12)

#### 间距
- 思考过程与回复内容间距：8px (1单位)
- 时间线步骤间距：9.6px (1.2单位)
- 时间线左侧padding：18px (2.25单位)

#### 字体
- 思考过程折叠：caption (12px)
- 步骤标题：caption (12px)，加粗
- 步骤描述：body2 (14px)

#### 动画
- 展开/收起：Collapse组件，200ms
- 占位符打字效果：55ms间隔
- 进行中图标：呼吸动画，1.2s循环

### 6. 边界情况处理

#### 无思考过程
- 如果timeline为空且trace为空且不在active状态
- 不显示ThinkingProcess组件

#### 只有占位符
- 显示默认占位符："理解问题"
- 应用打字机动画效果

#### 错误状态
- 步骤标记为error
- 使用红色主题
- 显示"失败"标签

#### 长文本处理
- 折叠状态：单行，超出使用省略号
- 展开状态：完整显示，自动换行

### 7. 兼容性考虑

#### 响应式设计
- 移动端：思考过程组件宽度100%
- 桌面端：保持左对齐，最大宽度80%

#### 现有功能保留
- 实时流式更新
- 引用内联显示（CitationInline）
- 引用列表显示（CitationList）
- Markdown渲染
- 句子级引用映射

### 阶段2：修改ChatMessageList

2. 移除"AI"标签

5. 确认CitationList保留（因为在内容末尾）

### 阶段3：样式优化
1. 调整颜色和字体
2. 优化动画效果
3. 测试响应式布局
4. 确保视觉一致性

### 阶段4：测试验证
1. 测试折叠/展开交互
2. 测试实时更新
3. 测试各种状态（进行中、完成、错误）

## 成功标准
1. AI消息无背景、无边框，直接显示在页面上
2. 思考过程折叠时只显示单行文字
3. 思考过程展开时显示简洁的时间线
4. 用户消息保持原有样式
5. 所有现有功能正常工作
6. 动画流畅自然
7. 响应式布局正常

## 风险和注意事项
1. 占位符动画逻辑需要仔细移植
2. 实时更新的状态同步需要测试
3. 时间线构建逻辑较复杂，需要保持一致性
4. 需要确保引用功能不受影响
5. 移除Paper后需要确保消息对齐正确
