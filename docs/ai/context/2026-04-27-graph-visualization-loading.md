# 2026-04-27 知识图谱页面加载范围调整

## 背景

知识图谱页面此前固定请求 `limit=50`，后端按关系边 `LIMIT` 后只补这些边两端的节点。实际 Neo4j 中已有更多图谱内容时，前端会只展示部分内容，并且统计值也是“本次返回数量”，容易被误解为完整图谱规模。

## 决策

- 前端 graph 页面默认请求上限提高到 `1000` 条关系。
- 前端 hook、领域 API、后端 endpoint、后端 service 的默认 `limit` 统一为 `1000`。
- 后端直连 Neo4j 查询改成先查同一 `group_id` 下的全部节点，再查按 `edge.created_at DESC, edge.uuid ASC` 稳定排序后的关系边。
- 返回节点不再只依赖关系两端，因此没有关系的孤立节点也会出现在图谱数据中。

## 取舍

- 当前本地数据规模约数百节点和关系，一次性返回 `1000` 条关系可以满足完整展示，性能风险可控。
- `limit` 仍控制关系数量，避免未来图谱快速增长后一次性渲染过多边导致前端卡顿。
- 孤立节点纳入节点列表后，如果没有边，会在可视化中作为独立节点出现；后续如果节点过多，需要增加搜索、筛选或分层加载。

## 后续建议

- 后端 stats 应进一步区分 `returned_*` 和 `total_*`，避免分页或上限加载时统计语义不清。
- 如果图谱超过数千节点，应优先引入搜索、邻域展开、社区聚类或按主题过滤，而不是直接展示全量图。
