# 本地与 Docker 端口规则

日期：2026-04-27

## 背景

本地临时启动时曾把当前修改版前端放到 `5174/5180/5181`，后端放到 `8012`。这会造成三个问题：

- 浏览器入口和 Docker 暴露入口不一致
- 后端 CORS 与前端真实端口不一致
- Docker、后端和用户访问地址无法统一定位当前服务

因此项目需要固定联调端口规则。

## 端口规则

- 前端默认端口固定为 `5173`
- 后端默认端口固定为 `8000`
- Docker Compose 默认映射保持 `FRONTEND_PORT=5173`、`BACKEND_PORT=8000`
- 前端默认访问后端 `VITE_API_BASE_URL=http://localhost:8000`
- 后端 CORS 必须放行：
  - `http://localhost:5173`
  - `http://127.0.0.1:5173`

## 操作原则

- 如果 `5173/8000` 已被占用，先检查是否已有项目 Docker 服务在运行。
- 不要为了临时方便把交付入口改到 `5174/5180/5181`。
- 备用端口只能用于短时排障，排障后必须停止，不能作为用户验收入口。
- 当前项目本地验收入口统一为 `http://127.0.0.1:5173`。

## 影响

后续设置页、chat 页面、memories 页面验证都以 `5173 -> 8000` 为准。端口变更不应写入代码或文档作为默认值。
