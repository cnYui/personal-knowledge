## 3. ✅ 插入知识图谱

### 前端操作
用户在记忆详情对话框中点击"加入知识图谱"按钮

### 后端实现
- **API**: `POST /api/memories/{id}/add-to-graph`
- **HTTP 状态**: 202 Accepted (异步处理)
- **处理流程**:
  1. 记忆 ID 加入 asyncio.Queue
  2. GraphitiIngestWorker 从队列取出任务
  3. 调用 Graphiti SDK 处理内容
  4. 使用 StepFun API 进行 LLM 分析
  5. 使用本地 embedding 模型生成向量
  6. 将实体和关系存储到 Neo4j
  7. 更新 SQLite 中的图谱状态

### 数据库操作
**SQLite**:
```sql
UPDATE memory 
SET graph_status = 'added',
    graph_episode_uuid = ?,
    graph_added_at = ?
WHERE id = ?
```

**Neo4j**:
- 创建 Episodic 节点
- 创建 Entity 节点
- 创建 RELATES_TO 关系
- 建立全文索引

### 验证结果
- ✅ 记忆成功加入异步处理队列
- ✅ 后台 worker 正常处理
- ✅ 数据正确存储到 Neo4j 图数据库
- ✅ SQLite 中的状态字段正确更新
- ✅ 可以通过 episode_uuid 追踪图谱数据
- ✅ 前端轮询自动更新状态显示

### 状态追踪
- `not_added`: 未添加
- `pending`: 处理中
- `added`: 成功添加
- `failed`: 处理失败

## 4. ✅ 搜索功能

### 前端操作
用户在搜索框输入关键词 → 实时过滤记忆列表

### 后端实现
- **API**: `GET /api/memories?keyword={keyword}`
- **数据库查询**:
```sql
SELECT * FROM memory 
WHERE title LIKE '%keyword%' 
   OR content LIKE '%keyword%'
ORDER BY updated_at DESC
```

### 搜索特性
- ✅ 支持英文关键词搜索
- ✅ 支持中文关键词搜索
- ✅ 同时搜索标题和内容
- ✅ 大小写不敏感
